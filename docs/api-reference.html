Quick Start — Express Middleware

import express from 'express';
import { x402Middleware } from 'casper-forge-x402';

const app = express();

// Protect any route with x402 CSPR payment
app.get('/api/analyze-rwa',
  x402Middleware({
    amount: '1000000000',  // 1 CSPR in motes
    wallet: 'account-hash-0123...',
    description: 'RWA compliance analysis',
    chain: 'casper-test',   // or 'casper' for mainnet
  }),
  (req, res) => {
    res.json({ result: 'analysis data', paid: true });
  }
);

app.listen(3000);

HTTP Protocol

Step 1: Client Request (No Payment)

GET /api/analyze-rwa HTTP/1.1
Host: api.casper-forge.io

Step 2: Server Response (402 Payment Required)

HTTP/1.1 402 Payment Required
WWW-Authenticate: x402
X-PAYMENT: {"version":1,"amount":"1000000000","asset":"CSPR",...}

{
  "error": "Payment Required",
  "x402Version": 1,
  "accepts": {
    "amount": "1000000000",
    "asset": "CSPR",
    "chain": "casper-test",
    "wallet": "account-hash-...",
    "paymentReference": "pay_abc123..."
  },
  "instructions": {
    "1": "Create a Casper transfer with the specified amount",
    "2": "Sign the transfer with your Ed25519 private key",
    "3": "Send payment payload in X-PAYMENT-SIGNATURE header",
    "4": "Retry this request with the payment header"
  }
}

Step 3: Client Pays (With Payment Header)

GET /api/analyze-rwa HTTP/1.1
Host: api.casper-forge.io
X-PAYMENT-SIGNATURE: {"payload":"...","signature":"..."}

Step 4: Server Delivers (200 OK)

HTTP/1.1 200 OK

{
  "result": "analysis data",
  "paid": true,
  "settlement": { "deploy_hash": "0x...", "cost": "1 CSPR" }
}
Why FORGE? Unlike competitor x402 implementations that use header-only mock validation, Casper Forge performs real on-chain CSPR transfers via the Casper deploy API. Every payment is cryptographically verified and settled on the Casper blockchain.
